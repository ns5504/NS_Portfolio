01 · Excel · Tableau · Retail Analytics

Luxury Fashion Retail Performance Dashboard

Goal

Analyze sales performance, pricing discipline, seasonality, and return behavior across categories and brands to support data-driven merchandising and planning in a luxury retail environment.

What I Did

  • Conducted rigorous data cleaning and validation in Excel, standardizing season, category, and return status fields.
  • Developed helper metrics (markdown amount, return flags, season order) and built pivot-based analyses.
  • Evaluated category, brand, and season-level performance to identify commercial priorities.
  • Analyzed return rates and drivers to highlight risks to customer experience and operations.

Key Insights

  • Outerwear was the highest-revenue category, supporting prioritization in assortment planning.
  • Shoes, Dresses, and Bottoms showed the highest markdown intensity; Outerwear maintained strongest pricing discipline.
  • Summer generated peak sales - Zara led overall, Mango strongest in Fall, H&M in Summer.
  • "Changed Mind" and "Size Issue" were dominant return reasons - improved presentation could reduce costs.

02 · SQL · Tableau · Policy Analytics

US Alternative Fuel & EV Adoption Dashboard

Problem

Support transportation planning by understanding how EVs and alternative fuels are adopted across U.S. states, and identify where charging infrastructure investment should be prioritized.

My Approach

  • Cleaned and standardized state-level registration data across fuel types.
  • Calculated EV adoption rate (% of total vehicles) for fair cross-state comparisons.
  • Ranked top and bottom states by adoption and examined regional patterns.
  • Built an interactive Tableau dashboard for non-technical stakeholders.

Key Findings

  • EV adoption is concentrated in a small set of coastal and urban states.
  • California leads by both volume and adoption rate.
  • E85 is the only alternative fuel with broad national presence.
  • Hydrogen vehicles remain niche and concentrated in California.
  • Several states lag significantly, indicating infrastructure and policy gaps.

03 · Python · SQL · Data Validation

Scientific Data Migration Validation at NSLS-II

Overview

At Brookhaven National Laboratory's National Synchrotron Light Source II, I validated the migration of 10+ petabytes of scientific data from MongoDB (NoSQL) to PostgreSQL (SQL) within the Tiled data management framework.

My Role

  • Validated correctness of the large-scale NoSQL → SQL migration.
  • Compared query outputs across legacy and new systems.
  • Monitored run counts, metadata consistency, and migration logs.
  • Built visual summaries to communicate migration health.

Methods

  • Compared daily run counts (2021–2025) across both systems.
  • Verified run-type distributions (FlyPlan 1D, FlyPlan 2D, others).
  • Parsed logs and categorized warnings vs. errors using Matplotlib.

Key Findings

  • Run-type distributions matched within <0.5%.
  • 64.5% of runs completed with no issues.
  • ~35% completed with warnings but no data loss.
  • Only 0.1% failed — strong system reliability confirmed.

Why It Matters

Confirmed the SQL-backed system preserves data integrity at scale, supporting faster access and a planned facility-wide rollout across NSLS-II beamlines.

04 · Python · NLP · BERT / RoBERTa

Assessing the Impact of ChatGPT in Education

Overview

Analyzed 2,086 Reddit posts from education-focused subreddits (Dec 2022 – Aug 2024) to understand perceptions of ChatGPT's impact on learning, teaching, and academic integrity.

Methods

  • Collected posts via PullPush API (keyword "ChatGPT") across multiple subreddits.
  • Cleaned spam/duplicates, filtered to English, removed incomplete entries.
  • Created a validation set of 200 labeled posts with refined annotation guidelines.
  • Sentiment: VADER, BERT, RoBERTa, Logistic Regression + GPT comparisons.
  • Topics: LDA, CorEx, BERTopic.

Key Findings

  • Strong concern around plagiarism and cheating.
  • Fear of reduced critical thinking due to over-reliance on AI tools.
  • Educators reported difficulty assessing authentic learning.
  • Topics surfaced ethics, unequal access, and policy uncertainty.

Model Insights

  • Outputs varied significantly by model — manual labels skewed more negative than GPT.
  • GPT-3.5 / GPT-4 with reasoning handled nuanced cases more consistently.
